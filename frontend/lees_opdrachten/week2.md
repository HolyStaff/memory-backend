# 2
Ja, ik ben het eens met de schrijver van het artikel dat mobile-first de betere aanpak is. 
Een belangrijk argument uit het artikel waar ik het mee eens ben, is dat websites van nature al responsief zijn zonder CSS, 
waardoor je met mobile-first meteen een goed werkend basisontwerp hebt. 
Door te beginnen met de mobiele versie bouw je stap voor stap op naar grotere schermen zonder dat je onnodige of dubbele code hoeft te schrijven, 
wat vaak wel gebeurt bij desktop first development. 
Dat maakt mobile-first efficienter en zorgt voor cleaner, beter onderhoudbare code.
